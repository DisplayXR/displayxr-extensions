## What this repo is FOR

A **public, OpenXR-compatible distribution surface** for the
DisplayXR runtime's custom extensions. Consumed by:

- Third-party app developers writing OpenXR apps for DisplayXR.
- Workspace-controller authors (OEMs / verticals) implementing
  `XR_EXT_spatial_workspace`.
- The DisplayXR Shell + Unity / Unreal plugins as a public header
  reference (so they don't pin against runtime-internal paths).

Authenticated via the `displayxr-publish-bot` GitHub App (org
secrets `DISPLAYXR_APP_ID` + `DISPLAYXR_APP_PRIVATE_KEY`).

## Releasing

**This repo does not have a release flow.** There are no tags, no
GitHub Releases, no installer artifacts. Consumers either:
- Submodule the repo (`git submodule add ... external/displayxr-extensions`)
  and update the submodule pointer when they want newer extensions.
- Copy individual headers into their project's include path.

Extensions evolve via their `spec_version` field inside each header,
which the runtime advertises through
`xrEnumerateInstanceExtensionProperties`. Consumers check
`extension.specVersion` at runtime to gate features. This is the
OpenXR canonical pattern â€” no separate package versioning needed.

## Not in the runtime's `versions.json` auto-bump matrix

The runtime maintains a
[`versions.json`](https://github.com/DisplayXR/displayxr-runtime/blob/main/versions.json)
that pins the **bundled stack** (runtime, shell, leia-plugin, mcp,
demos) for the dev orchestrator and the meta-installer. **This
extensions repo is NOT a field in that matrix** â€” it has no tags,
no installers, no runtime version to pin. It's a header mirror,
nothing more.

Spec: [`displayxr-runtime/docs/specs/runtime/versions-json-autobump.md`](https://github.com/DisplayXR/displayxr-runtime/blob/main/docs/specs/runtime/versions-json-autobump.md).
